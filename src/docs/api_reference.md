============================================================
                  مرجع API (API Reference)
============================================================

نام پروژه: my_bot_project
نسخه API: 1.0.0
پروتکل: HTTP/HTTPS
فرمت داده: JSON

------------------------------------------------------------
                    معرفی کلی
------------------------------------------------------------

این مستندات شامل توضیح کامل تمام Endpointهای موجود در سیستم
می‌باشد. APIها عمدتاً برای ارتباط با وب‌هوک و سرویس‌های خارجی
طراحی شده‌اند.

نکته مهم: اکثر عملیات از طریق ربات تلگرام انجام می‌شود و APIها
عمدتاً برای عملیات‌های مدیریتی و یکپارچه‌سازی با سیستم‌های خارجی
استفاده می‌شوند.

------------------------------------------------------------
                    آدرس پایه (Base URL)
------------------------------------------------------------

توسعه: http://localhost:8080/api/v1
تولید: https://api.mybot.com/api/v1

------------------------------------------------------------
                    احراز هویت (Authentication)
------------------------------------------------------------

بیشتر Endpointها نیاز به احراز هویت با استفاده از توکن Bearer
دارند:

Authorization: Bearer <your_api_token>

توکن API را می‌توانید از پنل مدیریت دریافت کنید.

------------------------------------------------------------
                    کدهای وضعیت (Status Codes)
------------------------------------------------------------

کدهای وضعیت استاندارد:

200 OK          - درخواست با موفقیت انجام شد
201 Created     - منبع جدید ایجاد شد
400 Bad Request - درخواست نامعتبر
401 Unauthorized - احراز هویت ناموفق
403 Forbidden   - دسترسی غیرمجاز
404 Not Found   - منبع مورد نظر یافت نشد
422 Unprocessable Entity - داده‌های ارسالی نامعتبر
429 Too Many Requests - تعداد درخواست‌ها بیش از حد مجاز
500 Internal Server Error - خطای سرور

------------------------------------------------------------
                    پاسخ‌های استاندارد
------------------------------------------------------------

پاسخ موفق:

{
    "status": "success",
    "data": { ... },
    "message": "عملیات با موفقیت انجام شد",
    "timestamp": "2024-01-01T12:00:00Z"
}

پاسخ خطا:

{
    "status": "error",
    "error": {
        "code": "VALIDATION_ERROR",
        "message": "داده‌های ورودی نامعتبر هستند",
        "details": {
            "field": "email",
            "message": "ایمیل نامعتبر است"
        }
    },
    "timestamp": "2024-01-01T12:00:00Z"
}

============================================================
                  1. وب‌هوک (Webhook)
============================================================

------------------------------------------------------------
1.1. دریافت وب‌هوک
------------------------------------------------------------

مسیر: POST /webhook

توضیح: این Endpoint برای دریافت رویدادهای ارسالی از سرویس‌های
خارجی (مثل درگاه پرداخت) استفاده می‌شود.

بدنه درخواست:

{
    "event": "payment.success",
    "data": {
        "transaction_id": "txn_123456789",
        "order_id": 1001,
        "amount": 150000,
        "reference": "ref_987654321",
        "status": "completed",
        "timestamp": "2024-01-01T12:00:00Z"
    },
    "signature": "abcdef1234567890"
}

پاسخ موفق:

{
    "status": "success",
    "message": "Webhook received successfully",
    "data": {
        "processed": true,
        "transaction_id": "txn_123456789"
    }
}

کدهای خطا:

400 - بدنه درخواست نامعتبر
401 - امضای نامعتبر
404 - رویداد نامشخص
422 - داده‌های رویداد نامعتبر

============================================================
                  2. کاربران (Users)
============================================================

------------------------------------------------------------
2.1. دریافت اطلاعات کاربر
------------------------------------------------------------

مسیر: GET /users/{user_id}

توضیح: دریافت اطلاعات کامل یک کاربر بر اساس شناسه.

پارامترهای مسیر:

- user_id: شناسه کاربر (integer, required)

پارامترهای Query:

- include_orders: شامل سفارشات (boolean, default: false)
- include_feedback: شامل بازخوردها (boolean, default: false)

پاسخ موفق:

{
    "status": "success",
    "data": {
        "id": 1,
        "telegram_id": 123456789,
        "username": "test_user",
        "first_name": "Test",
        "last_name": "User",
        "is_active": true,
        "level": "gold",
        "points": 1000,
        "created_at": "2024-01-01T10:00:00Z",
        "updated_at": "2024-01-15T14:30:00Z"
    }
}

کدهای خطا:

404 - کاربر یافت نشد

------------------------------------------------------------
2.2. جستجوی کاربران
------------------------------------------------------------

مسیر: GET /users

توضیح: جستجوی کاربران با استفاده از فیلترهای مختلف.

پارامترهای Query:

- search: عبارت جستجو (string, optional)
- level: سطح کاربر (string, optional) - gold, silver, bronze, normal
- is_active: وضعیت فعال بودن (boolean, optional)
- limit: تعداد نتایج (integer, default: 20, max: 100)
- offset: نقطه شروع (integer, default: 0)
- sort_by: ترتیب بر اساس (string, default: "created_at")
- sort_order: جهت ترتیب (string, default: "desc") - asc, desc

پاسخ موفق:

{
    "status": "success",
    "data": {
        "items": [
            {
                "id": 1,
                "telegram_id": 123456789,
                "username": "test_user",
                "first_name": "Test",
                "last_name": "User",
                "level": "gold",
                "is_active": true,
                "created_at": "2024-01-01T10:00:00Z"
            },
            {
                "id": 2,
                "telegram_id": 987654321,
                "username": "user2",
                "first_name": "Second",
                "last_name": "User",
                "level": "silver",
                "is_active": true,
                "created_at": "2024-01-02T10:00:00Z"
            }
        ],
        "total": 10,
        "limit": 20,
        "offset": 0
    }
}

کدهای خطا:

400 - پارامترهای نامعتبر

------------------------------------------------------------

مسیر: POST /coupons/validate

توضیح: اعتبارسنجی یک کد کوپن قبل از استفاده.

بدنه درخواست:

{
    "code": "SUMMER2024",
    "user_id": 1,
    "order_amount": 75000
}

    "status": "success",

============================================================
                  نرخ‌های محدودیت (Rate Limits)
============================================================

بیشتر Endpointها دارای محدودیت نرخ درخواست هستند:

- کاربران عمومی: ۱۰۰ درخواست در دقیقه
- ادمین‌ها: ۵۰۰ درخواست در دقیقه
- وب‌هوک‌ها: ۱۰۰۰ درخواست در دقیقه

هدرهای مربوط به Rate Limit:

- X-RateLimit-Limit: تعداد مجاز درخواست‌ها
- X-RateLimit-Remaining: تعداد باقی‌مانده
- X-RateLimit-Reset: زمان بازنشانی (Unix timestamp)
